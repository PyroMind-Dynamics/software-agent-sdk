## Prerequisites

- You have [created a JupyterLab instance](/docs/jupyterlab/how-to-use) and [connected to JupyterLab](/docs/jupyterlab/how-to-use#step-3-connect-to-instance)
- You completed [SDK installation](/docs/sdk/installation) in the JupyterLab terminal:

  ```bash
  python3 -m venv .venv
  source .venv/bin/activate
  pip install pyromind-sdk
  ```
- You are familiar with [System nodes](/docs/studio/system-nodes)
- You have set the `PYROMIND_API_KEY` environment variable
 ```bash
  export PYROMIND_API_KEY="your-api-key"
  ```

## Why custom nodes

**System nodes** cover common data, training, and inference scenarios. When business logic is more specialized, such as custom data preprocessing or output post-processing, you can package Python functions as **custom nodes** and use them in Studio with drag-and-drop wiring just like built-in nodes.

Typical scenarios:

| Scenario | Example |
|------|------|
| Data preprocessing | Filter, transform, or augment samples using business-specific rules |
| Output processing | Format model outputs, validate structure, or write to Storage |

## End-to-end flow

Complete all steps below in JupyterLab:

```
Write Python function in JupyterLab -> Generate YAML with CLI -> Validate and register with SDK -> Use on Studio canvas
```

---

## Step 1: Write a node function in JupyterLab

Open JupyterLab, create `/workspace/nodes/` in the left file panel, and create `greet.py`:

```python
# /workspace/nodes/greet.py
def greet(name: str = "World") -> dict:
    """Return a greeting message."""
    return {"message": f"Hello, {name}!"}
```

<Note>
  - Open a JupyterLab terminal from **File -> New -> Terminal**
  - Use the built-in editor or `vim`/`nano` in terminal to create Python files
  - Keep function parameters and return types explicit so input/output ports can be generated automatically
</Note>

---

## Step 2: Generate YAML with CLI

Run this command in the JupyterLab terminal:

```bash
python -m pyromind_sdk.python_function_to_yaml_cli \
  /workspace/nodes/greet.py greet \
  --node-name GreetNode \
  --output /workspace/nodes/greet.yaml
```

<Note>
  Store YAML files under `/workspace/nodes/`; otherwise the server cannot read them during registration. In YAML, `python_code` must point to the actual `.py` file path.
</Note>

After execution, `greet.yaml` is generated under `/workspace/nodes/`:

```yaml
name: GreetNode
description: Return a greeting message.
display_name: GreetNode
base_class: PodExecutionNode
python_code: /workspace/nodes/greet.py
function_name: greet
python_command: python3
parameters:
  - name: name
    dtype: STRING
    type: input
    required_type: optional
  - name: message
    dtype: STRING
    type: output
```

---

## Step 3: Validate YAML

Validate the generated YAML in the JupyterLab terminal:

```bash
python -m pyromind_sdk.tests.test_yaml_nodes /workspace/nodes/greet.yaml --verbose
```

If validation succeeds, the output includes node class name and input/output port information.

---

## Step 4: Register to Studio

Create `register.py` in the JupyterLab terminal and run it:

```python
# /workspace/nodes/register.py
from pyromind_sdk.nodes.yaml_loader import load_nodes_from_yaml
from pyromind_sdk.nodes.node_validator import validate_node_class
from pyromind_sdk import PyroMindAPIClient

# 1. Validate
nodes = load_nodes_from_yaml("/workspace/nodes/greet.yaml")
node_class = nodes["GreetNode"]
validate_node_class(node_class, "GreetNode")

# 2. Register
client = PyroMindAPIClient()
client.studio.create_node(yaml_path="/workspace/nodes/greet.yaml")
print("Node registration succeeded")
```

Run in terminal:

```bash
python /workspace/nodes/register.py
```

After success, `GreetNode` appears in the Studio node library.

---

## Step 5: Use on the Studio canvas

1. Open [Training Studio](https://console.pyromind.ai/studio), then create or open a workflow.
2. Search for `GreetNode` in the node library.

![Search GreetNode](/resource/studio/imgs/studio-custom-nodes-search.png)

3. Drag it onto the canvas and connect input/output ports with upstream and downstream nodes.

![Drag onto canvas](/resource/studio/imgs/studio-custom-nodes-workflow.png)

4. Fill in the `name` parameter, save, and run the workflow.

## Common issues

- **Custom node not visible in node library**: Confirm `create_node` executed successfully
- **Parameter error during execution**: Check whether YAML port types match upstream output types
- **Need to update node logic**: After updating `.py` or `.yaml`, run `client.studio.reload_nodes(node_name="YourNode")` to apply changes
