## Prerequisites

- You have completed [Model validation](/docs/studio/model-validation) and selected a production checkpoint
- You understand the [Inference](/docs/basic/inference) console or [SDK inference jobs](/docs/sdk/inference)

## Deployment options overview

| Method | Best for |
|------|----------|
| Inference service | Always-on API endpoints for business systems (recommended) |
| SDK | Programmatic job creation and automation pipelines |

After training, model checkpoints are usually stored in Storage or the workflow's `output_path`. Once validated, you can deploy directly as an Inference service.

## Deploy to Inference

1. Confirm model files are ready in Storage.
2. Create an inference job in the [Inference](/docs/basic/inference) console, or use the SDK:

```python
job_id = client.inference.create(
    InferenceJobRequest(
        model_path="/path/to/checkpoint",
        inference_framework=frameworks[0],
        resources=ResourceConfig(cpu="8", memory="32Gi", gpu="1"),
        name="my-deployed-model"
    )
)
```

3. Wait until job status is `running`, then get `endpoint_url`.
4. Send requests to the endpoint to complete production validation.

## Production readiness checklist

- [ ] Inference latency and throughput meet business requirements
- [ ] Input/output format matches API documentation
- [ ] Monitoring and logging are integrated
- [ ] Rollback plan (previous checkpoint retained) is prepared
