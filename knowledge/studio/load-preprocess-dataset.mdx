## Prerequisites

- You have [signed up and logged in](/docs/basic/signup_login) and are familiar with [Studio](/docs/studio/how-to-use)
- Your dataset has been uploaded to [distributed storage](/docs/basic/storage)

## Import the dataset processing workflow

Use the official **Dataset Processing** workflow to load data, convert formats, and validate inputs:

1. Download the workflow: <a href="/resource/studio/jsons/dataset-processing.json" target="_blank" rel="noreferrer">dataset-processing</a>
2. Drag the JSON file into the Studio canvas.
3. Configure each node as described below.

![Dataset processing workflow](/resource/studio/imgs/Dataset-Processing.png)

## Workflow nodes

| Node | Description |
|------|------|
| Clone Dataset | Pull a dataset from HuggingFace or similar sources into a local cache path |
| Join Path | Join the dataset root directory with a file subpath |
| Dataset to JSONL | Convert formats such as Parquet into JSONL |
| Dataset Kind Config Builder (Text Only) | Configure field mappings and output `dataset_kind_config` |
| Dataset Config Builder | Combine data paths and `dataset_kind_config` to build full `dataset_config` |
| Dataset Validator | Validate dataset format and generate a preview report |

## Prepare dataset format

Before training, ensure your dataset matches the target task format. For LLM supervised fine-tuning, each sample typically includes **question** and **answer** fields:

- **Question**: `"Natalia sold clips to 48 of her friends in April, then sold half as many in May. How many clips did Natalia sell in total in April and May?"`
- **Answer**: Complete response text with reasoning steps and final result

For DPO, include an additional **rejected** field (the rejected answer). For VLM/GUI Agent tasks, use **Dataset Kind Config Builder (Vision)** and configure fields such as `image_field`.

## Configure dataset nodes

Field mappings and data paths are configured in separate nodes:

**Dataset Kind Config Builder (Text Only)** - Configure field mapping, and connect the output to `dataset_kind_config` on **Dataset Config Builder**:

| Field | Description |
|--------|------|
| `user_prompt_field` | User input field (default in example workflow: `question`) |
| `assistant_response_field` | Model response field (default in example workflow: `answer`) |
| `system_prompt_field` | System prompt field (optional) |
| `rejected_field` | Rejected-answer field for DPO tasks (empty in the Dataset Processing workflow; configure later in DPO training) |

**Dataset Config Builder** - Configure data paths:

| Field | Description |
|--------|------|
| `train_data_path` / `val_data_path` | JSONL paths for train and validation datasets |
| `dataset_kind_config` | Connected from Dataset Kind Config Builder |
| `dataset_extra_config` | Connected from Dataset Extra Config Builder (used in training workflows) |

## Preprocessing and format conversion

1. In **Clone Dataset**, choose a dataset (for example, `openai/gsm8k`) and set the cache directory (default: `/workspace/datasets/`).
2. In **Join Path**, enter the data file subpath (for example, `main/train-00000-of-00001.parquet`).
3. If source data is not JSONL, convert it with **Dataset to JSONL**.
4. Connect the joined path to `train_data_path` in **Dataset Config Builder**.

<Note>
Specific preprocessing node names and parameters vary by task type. After importing official workflows, refer to each node's property panel for details.
</Note>
