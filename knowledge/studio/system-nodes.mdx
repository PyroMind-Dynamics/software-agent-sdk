## Prerequisites

- You are familiar with basic canvas operations in [Studio](/docs/studio/how-to-use)

## What are system nodes

**System nodes** are built-in standard nodes in Studio. They encapsulate core capabilities such as data loading, model management, training configuration, training execution, and inference. By connecting system nodes in a data flow, you can build complete training and inference workflows without writing low-level scripts.

Each node includes:

- **Input/output ports** - Transfer datasets, model paths, config objects, and more between nodes
- **Property panel** - Configure paths, hyperparameters, resources, and other settings on the right panel
- **nodeType** - Node type identifier (visible when importing workflow JSON)

<Note>
In addition to system nodes, you can extend the node library with [Custom nodes](/docs/studio/custom-nodes). Config Builder nodes are usually pre-registered by the platform and can be used directly like system nodes.
</Note>

## Node categories

### Data and resources

| Node | nodeType | Description |
|------|----------|------|
| Clone Dataset | `CloneAndCacheDataset` | Pull and cache datasets from HuggingFace and similar sources to a local path |
| Clone Model | `CloneAndCacheModel` | Pull and cache pretrained models to a local path |
| Join Path | `PathJoinNode` | Join a base path and subpath |
| Dataset to JSONL | `DatasetToJsonlNode` | Convert formats such as Parquet to JSONL |
| Dataset Validator | `DatasetValidatorNode` | Validate dataset format and generate an HTML preview |
| Load Image | `LoadImage` | Load a single image for VLM/GUI inference |
| Load LoRA Adapter | `LoadLoRAAdapter` | Load LoRA adapters for LoRA-enabled inference |

### Config builders

| Node | nodeType | Description |
|------|----------|------|
| Dataset Config Builder | `DatasetConfigBuilderNode` | Build dataset YAML config (data path, `dataset_kind_config`, `dataset_extra_config`) |
| Dataset Kind Config Builder (Text Only) | `DatasetConfigBuilderTextNode` | Text-task field mapping; outputs `dataset_kind_config` |
| Dataset Kind Config Builder (Vision) | `DatasetConfigBuilderVisionNode` | Multimodal field mapping (includes `image_field`); outputs `dataset_kind_config` |
| Dataset Extra Config Builder | `DatasetExtraConfigBuilderNode` | Sequence length, sample caps, collator entry, and more |
| Model Config Builder | `ModelConfigBuilderNode` | Model path (`model_path`) and model type (`model_type`) |
| Lora Config Builder | `LoraConfigBuilderNode` | LoRA rank, dropout, target modules, excluded modules |
| Merge LoRA | `ModelMergeLoraNode` | Merge LoRA adapters into base model weights and output a standalone model directory |
| Training Config Builder | `TrainingConfigBuilderNode` | Learning rate, batch size, epochs, save/eval steps, and more |
| Accelerate Config Builder | `AccelerateConfigBuilderNode` | Distributed training and ZeRO stage settings |
| Wandb Config Builder | `WandbConfigBuilderNode` | WandB experiment tracking configuration |
| Reward Item Builder | `RewardItemBuilderNode` | Single reward entry function, name, and weight |
| Reward Config Builder | `RewardConfigBuilderNode` | Combine multiple reward items |
| GRPO Training Extra Config Builder | `GRPOTrainingExtraConfigBuilderNode` | Group size, temperature, generation length, and more |
| Metrics Config Builder | `MetricsConfigBuilderNode` | Metric entry and name for Benchmark workflows |

### Training execution

| Node | nodeType | Description |
|------|----------|------|
| SFT Training | `ModelTrainSFTNode` | Supervised fine-tuning |
| DPO Training | `ModelTrainDPONode` | Direct preference optimization |
| GRPO Training | `ModelTrainGRPONode` | GRPO reinforcement learning |

### Inference and evaluation

| Node | nodeType | Description |
|------|----------|------|
| Inference (VLLM) | `VLLMInference` | Start a vLLM inference service for Benchmark or business calls |
| Benchmark By Api | `ModelEvalApiNode` | Batch-evaluate models via inference APIs and output metric reports |
| Validate Reward | `ModelValidateRewardNode` | Validate reward configuration and scoring logic |
| LLM Test | `TestLLMNode` | Send test prompts to inference endpoints and return outputs |
| Content Preview | `ContentPreview` | Preview text content, useful for reviewing **LLM Test** output |

### Utilities

| Node | nodeType | Description |
|------|----------|------|
| Anything to String | `AnythingToString` | Convert any output type into a string |
| Show Result | `ShowResultNode` | Display upstream node outputs |
| Preview Image | `PreviewImageEx` | Preview images to debug VLM/GUI workflows |

## Best practices

1. **Import examples first**: Start from official JSON and adjust paths/hyperparameters in the property panel. This is faster than wiring from scratch.
2. **Match port types**: Connect Config Builder string outputs to the corresponding config inputs on training nodes.
3. **Connect model path directly**: Connect `model_path` from **Clone Model** directly into `model_path` on **Model Config Builder**.
4. **Separate model and LoRA config**: Set `model_path` in **Model Config Builder** and LoRA params in **Lora Config Builder**. After training, merge with **Merge LoRA**.
5. **Run single-node smoke tests**: In complex workflows, validate data-loading or Dataset Validator outputs before launching training.
6. **Track training logs**: After configuring Wandb Config Builder, monitor loss and reward curves in WandB.
