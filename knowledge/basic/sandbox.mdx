## 前置条件

- 拥有有效的 PyroMind 账户并已 [登录控制台](https://console.pyromind.ai/dashboard)
- 了解 [账户配额](/zh/docs/basic/user_profile#第四步：查看配额与资源限制)
- （可选）已将自定义系统镜像（qcow2 格式）上传至 [分布式存储](/zh/docs/basic/storage)

## 典型流程

1. **创建实例** — 选择沙箱类型、配置资源与屏幕分辨率
2. **查看信息** — 获取 Sandbox ID 与 Web VNC URL
3. **远程访问** — 通过 Web VNC URL 在浏览器中操作沙箱桌面
4. **修改配置** — 调整标题、资源、屏幕分辨率等设置
5. **暂停 / 恢复 / 删除** — 按需管理实例生命周期

## 沙箱类型

Sandbox 支持多种系统类型，覆盖桌面操作系统、移动端模拟器等不同场景：

| 类型 | 说明 |
|------|------|
| OSWorld | 通用桌面操作系统沙箱，默认使用 Ubuntu 镜像，支持自定义 qcow2 系统镜像 |

<Note>
不同类型实际可用范围以控制台界面展示为准。OSWorld 等桌面类沙箱默认开启 Web VNC，可在浏览器中直接交互。
</Note>

## 第一步：进入 Sandbox

1. 登录 [PyroMind 控制台](https://console.pyromind.ai/dashboard)。
2. 在导航栏或产品列表中找到 **Sandbox** 入口并进入实例管理页面。

## 第二步：创建 Sandbox 实例

1. 在实例列表页面点击「Create / 创建」按钮，并在弹出的对话框中切换至 **Sandbox** 标签页。
2. 按表单依次填写：

   | 字段 | 说明 |
   |------|------|
   | Title | 实例名称，最多 100 个字符 |
   | Type | 沙箱类型，目前只支持 OSWorld |
   | Screen resolution | 屏幕分辨率，默认 `1920 × 1080` |
   | System image path | （可选）自定义系统镜像在 Storage 中的路径，留空则使用默认 Ubuntu 镜像 |
   | CPU | CPU 核数，最少 4 核 |
   | Memory (Gi) | 内存大小，单位 GiB，最少 8 Gi |

3. 确认无误后点击 **Create** 启动实例。

![Sandbox - 创建实例](/resource/sandbox/create.png)

<Note>
**资源比例要求**：CPU 与内存必须满足 **1 : 2** 的比例（例如 4 核对应 8 Gi，8 核对应 16 Gi）。不满足该比例的配置会被拒绝。

**自定义系统镜像**：`System image path` 需要指向 Storage 中已存在的 qcow2 镜像文件，例如 `/workspace/images/Windows-10-x64.qcow2`。沙箱启动后将以该镜像作为系统盘。
</Note>

## 第三步：查看实例信息

实例创建后，您可以在实例列表中看到名称、状态、资源配置等基本信息；点击实例可查看详细信息：

| 信息 | 说明 |
|------|------|
| Name | 创建时设置的实例名称 |
| Sandbox ID | 沙箱唯一标识，格式如 `sb-acd1f23fd4e9` |
| Web VNC URL | 浏览器访问的远程桌面地址，可一键复制 |

点击 Web VNC URL 即可在新标签页中打开远程桌面，进入沙箱桌面环境进行操作。

![Sandbox - 实例详情](/resource/sandbox/sandbox_info.png)

<Note>
Code 类型的沙箱不会暴露 Web VNC URL；OSWorld 等桌面类沙箱默认会附带 `autoconnect=true&resize=scale` 参数，连接后会自动适配窗口大小。
</Note>

## 第四步：修改实例配置

在实例列表或详情页面点击 **设置 / Edit** 按钮，可对已有实例进行编辑：

- 修改 Title（显示名称）
- 调整 Type（沙箱类型）
- 调整 Screen resolution（屏幕分辨率）
- 调整 System image path（自定义系统镜像路径）
- 调整 CPU / Memory 资源

确认修改后点击 **Update** 应用变更。

![Sandbox - 修改配置](/resource/sandbox/sandbox_edit.png)

<Note>
修改 CPU / 内存时同样需要满足 1 : 2 的比例约束。调整资源或系统镜像通常需要实例重启后生效。
</Note>

## 第五步：暂停、恢复与删除

| 操作 | 说明 |
|------|------|
| 暂停 | 临时停止沙箱运行，保留实例配置；暂停期间不会再产生计算用量 |
| 恢复 | 从暂停状态恢复运行，继续上一次的会话 |
| 删除 | 永久移除实例并释放占用的计算配额 |

在实例列表中找到对应实例的操作按钮即可执行上述操作。

## 常见问题

**Q：为什么创建实例时提示资源配置不合法？**
A：请检查 CPU 与内存是否满足 1 : 2 的比例，并确保 CPU ≥ 4、内存 ≥ 8 Gi，且不超过账户剩余配额。

**Q：Web VNC URL 打开后无法连接怎么办？**
A：请先确认实例已处于 running 状态；如果刚刚创建，请等待系统镜像加载完成后再尝试访问。

**Q：可以使用自己的 Windows / macOS 镜像吗？**
A：可以。先将 qcow2 镜像上传至 [分布式存储](/zh/docs/basic/storage) 或者 从JupyterLab中下载到挂载的storage 中，然后在创建或编辑实例时通过 `System image path` 指定该镜像。
