## What distributed Storage is

Distributed Storage is an S3/MinIO-compatible object storage service provided by the PyroMind platform. Its primary use cases include:

- **Cross-instance data sharing**: Share datasets, model weights, and output files across different JupyterLab or Studio tasks
- **Persistent data retention**: Mount data to the instance `/workspace` directory so data remains after instance pause or restart
- **Support for advanced features**: Features such as [SSH combined username login](/docs/jupyterlab/ssh-combined-login) require initialization scripts under `/workspace` in Storage

## Prerequisites

- You have a valid PyroMind account and have [signed in to the Console](https://console.pyromind.ai/dashboard)
- You understand your [storage quota](/docs/basic/user_profile)

## Step 1: Check storage usage

1. Sign in to the [PyroMind Console](https://console.pyromind.ai/dashboard).
2. Open the Storage management page or the storage usage section in [User profile](/docs/basic/user_profile).
3. Check used capacity, total capacity, and remaining space.

![EchoMind - Storage management](/resource/storage/imgs/storage_management.png)

## Step 2: Upload files

1. On the Storage management page, select the target directory or bucket.
2. Click **Upload** and select local files or folders.
3. Wait for upload completion and confirm files appear in the file list.

You can also drag and drop files directly into the Storage file browser.

![EchoMind - Upload files](/resource/storage/imgs/upload_file.png)

## Step 3: Browse and download files

1. Navigate to the target directory in the Storage file browser.
2. Click a file name to preview or view file details (if supported).
3. Select files and click **Download** to save them locally.

![EchoMind - File management](/resource/storage/imgs/file_management.png)

## Step 4: Delete files

1. Select files or folders you no longer need in the file list.
2. Click **Delete** and confirm the action.

> **Note:** Deletion cannot be undone. Proceed carefully.

![EchoMind - File management](/resource/storage/imgs/file_delete.png)

## Step 5: Mount Storage to JupyterLab instances

When you create or edit a JupyterLab instance, you can mount Storage to the instance `/workspace` directory:

1. In the [JupyterLab instance creation page](/docs/jupyterlab/how-to-use), select the Storage to mount in configuration.
2. Confirm the mount path (typically `/workspace`).
3. After startup, access `/workspace` through JupyterLab or SSH to read and write files in Storage.

After Storage is mounted, all files saved under `/workspace` are persisted to distributed Storage and will not be lost after instance restarts.




## Step 6: Access through your local terminal

You can access Storage content from your local terminal.

## Relationship with SSH combined login

The [SSH combined username login](/docs/jupyterlab/ssh-combined-login) feature requires the instance to have mounted Storage and read/write access to `/workspace`. Multi-user initialization scripts must be placed under `/workspace` in Storage and are executed automatically when the instance starts.
