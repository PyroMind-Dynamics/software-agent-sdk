EchoMind is the fully automated data crawling and model training service provided in the PyroMind Console. You can configure upstream model APIs, collection policies, and auto-training parameters. The platform handles data collection and triggers follow-on training workflows when conditions are met.

## Prerequisites

- You hold a valid PyroMind account and have [signed in to the Console](https://console.pyromind.ai/dashboard)
- You have reviewed [account quotas](/docs/basic/user_profile#step-4-view-quotas-and-resource-limits) and confirmed sufficient compute resources to create instances

## Typical workflow

| Phase | Description |
|-------|-------------|
| Create instance | Configure name, model API, execution mode, training parameters, and compute resources in the Console, then submit |
| Check status | Monitor runtime status and resource usage in the instance list |
| Pause / resume | Pause or resume collection and training tasks as needed |
| Clean up resources | Delete instances after tasks complete to release occupied quota |

## Step 1: Open EchoMind

1. Sign in to the [PyroMind Console](https://console.pyromind.ai/dashboard).
2. In the navigation bar or product list, find **EchoMind** and open the instance management page.

![EchoMind - Entry](/resource/echomind/imgs/echomind_entry.png)

## Step 2: Create an EchoMind instance

1. On the instance list page, click **Create instance**.
2. Complete the configuration items below, then confirm and submit.

### Basic information

| Setting | Description |
|---------|-------------|
| Instance name | Display name used to identify the EchoMind instance |

### Model API configuration

| Setting | Description |
|---------|-------------|
| **API Mode** | Upstream model API invocation mode |
| **API URL** | Service endpoint for the upstream model API |
| **Model Name** | Original model name used for data crawling |
| **AccessKey** | Access key required to call the upstream API |

### Execution mode

| Mode | Status | Description |
|------|--------|-------------|
| **Manual** (manual intervention mode) | Available | Upload an executable Shell script to [Storage](/docs/basic/storage) in advance; when data collection meets the trigger conditions, the system runs the script automatically to connect to your custom training workflow (see below) |
| **Automatic** | Coming soon | Will roll out incrementally as capabilities mature |

<Note>
The custom runtime script must be a Shell script (`.sh`) uploaded to the path configured in Storage. Before creating an instance, confirm the file exists at that path and has execute permissions.
</Note>

### Shell scripts and workflows in manual intervention mode

In manual intervention mode, your Shell script can invoke Python and use the [PyroMind SDK](/docs/sdk/training-workflow) to submit and run Studio workflows.

We recommend designing the workflow as a two-stage process: **data cleaning first, then training**:

| Stage | Description |
|-------|-------------|
| Data cleaning | Process data collected in the current round—for example, remove invalid samples, normalize fields, or split training sets |
| Training | Run model training on cleaned data and produce checkpoints or metric results |

Each time collected data reaches **Training Batch Size**, the script runs again and **the same workflow is invoked repeatedly**. Design your workflow for repeated execution with proper cleanup and isolation:

| Recommendation | Description |
|----------------|-------------|
| Clean intermediate artifacts | Before each training round, remove temporary files or stale intermediate results from the previous round |
| Isolate output paths | Use a separate output directory for each run; include a round number or timestamp in the path |
| Avoid overwriting in-use resources | Do not overwrite datasets, model checkpoints, or log files still referenced by downstream nodes |

For workflow design and node layout, see [Studio](/docs/studio/how-to-use) and [Load and preprocess datasets](/docs/studio/load-preprocess-dataset).

### Auto-training parameters

| Setting | Description |
|---------|-------------|
| **Training Batch Size** | Number of samples required per training batch; reaching this count triggers the Shell script and follow-on training workflow |
| **Trajectory Buffer Size** | Upper limit for trajectory data cache, used to cap total samples and prevent unbounded growth; when the limit is reached, the system removes the oldest data by generation time |
| **Training Rounds** | Total number of automatic training rounds to run |
| **Save Path** | Data storage path; the path must include `{{instanceId}}` to ensure data isolation between instances |

### Compute resources

| Setting | Description |
|---------|-------------|
| **CPU** | CPU resources required for the instance |
| **Memory** | Memory resources required for the instance |

When creating an instance, confirm that the selected resources do not exceed your [account quotas](/docs/basic/user_profile#step-4-view-quotas-and-resource-limits).

## Step 3: Check runtime status

After an instance is created, you can view the following in the instance list:

| Information | Description |
|-------------|-------------|
| Instance ID and name | Unique identifier and display name |
| Current status | Such as **pending**, **running**, **paused**, **stopped** |
| Resource usage | CPU, memory, and other resources consumed by the instance |

Click instance details to view full runtime logs and collection progress.

![EchoMind - Instance status](/resource/echomind/imgs/echomind_list.png)

## Step 4: Pause, resume, and delete

| Action | Description |
|--------|-------------|
| Pause | Temporarily stop collection and training tasks while keeping instance configuration and collected data |
| Resume | Continue running from paused state |
| Delete | Permanently remove the instance and release occupied compute quota |

Use the corresponding action buttons in the instance list or details page.

![EchoMind - Instance operations](/resource/echomind/imgs/echomind_instance_operate.png)

## Step 5: View collection results

After collection is complete, result files are typically saved in mounted Storage or the configured output directory. You can browse and download collected data through the [distributed Storage console](/docs/basic/storage).

![EchoMind - Crawling results](/resource/echomind/imgs/echomind_crawling_results.png)

## Step 6: View auto-training progress

When collected data reaches the configured **Training Batch Size**, EchoMind triggers the Shell script and starts the corresponding training workflow.

![EchoMind - Auto-training](/resource/echomind/imgs/echomind_auto_train.png)

On this page, you can view current data collection progress, collected data details, and training progress for each round.
